在 TypeScript/JavaScript 中，使用 **`const`声明的箭头函数** 是更现代和推荐的方式。下面是详细对比和推荐：

## 📊 对比表

| 特性                | `function`声明 | `const`箭头函数 | 推荐              |
| ------------------- | -------------- | --------------- | ----------------- |
| **变量提升**        | ✅ 会提升       | ❌ 不会提升      | ✅ `const`更安全   |
| **作用域**          | 函数作用域     | 块级作用域      | ✅ `const`更现代   |
| **`this`绑定**      | 动态 `this`    | 词法 `this`     | ✅ `const`更可预测 |
| **可重新赋值**      | ✅ 可重新定义   | ❌ 不可重新赋值  | ✅ `const`防止误改 |
| **简洁性**          | 较冗长         | 更简洁          | ✅ `const`         |
| **TypeScript 支持** | ✅ 完整         | ✅ 完整          | ✅ 两者都支持      |

## 📝 修改建议

### 推荐使用 `const`箭头函数：

```
// ✅ 推荐：使用 const 箭头函数
export const getTokenExpiresAt = (): number => {
  const expiresAt = uni.getStorageSync(TOKEN_EXPIRES_AT);
  if (!expiresAt) return 0;
  
  const num = Number(expiresAt);
  return isNaN(num) ? 0 : num;
};
```

### 原来的 `function`写法：

```
// ⚠️ 不推荐：使用 function 声明
export function getTokenExpiresAt(): number {
  const expiresAt = uni.getStorageSync(TOKEN_EXPIRES_AT);
  if (!expiresAt) return 0;
  
  const num = Number(expiresAt);
  return isNaN(num) ? 0 : num;
}
```

## 🎯 完整的 utils/cache.ts 示例（使用 const）

```
// utils/cache.ts
import { UserInfo } from "@/api/system/user";

// 存储键名常量
const TOKEN_KEY = "token";
const REFRESH_TOKEN_KEY = "refresh_token";
const TOKEN_EXPIRES_AT_KEY = "token_expires_at";
const USER_INFO_KEY = "user_info";
const PERMISSIONS_KEY = "permissions";

// Token 相关
export const getToken = (): string => {
  return uni.getStorageSync(TOKEN_KEY) || "";
};

export const setToken = (token: string): void => {
  uni.setStorageSync(TOKEN_KEY, token);
};

// 刷新 Token
export const getRefreshToken = (): string => {
  return uni.getStorageSync(REFRESH_TOKEN_KEY) || "";
};

export const setRefreshToken = (token: string): void => {
  uni.setStorageSync(REFRESH_TOKEN_KEY, token);
};

// Token 过期时间
export const getTokenExpiresAt = (): number => {
  const expiresAt = uni.getStorageSync(TOKEN_EXPIRES_AT_KEY);
  if (!expiresAt) return 0;
  
  const num = Number(expiresAt);
  return isNaN(num) ? 0 : num;
};

export const setTokenExpiresAt = (expiresAt: string | number): void => {
  const expiresAtStr = typeof expiresAt === 'number' 
    ? expiresAt.toString() 
    : expiresAt;
  uni.setStorageSync(TOKEN_EXPIRES_AT_KEY, expiresAtStr);
};

// 用户信息
export const getUserInfo = (): UserInfo | undefined => {
  const userInfoStr = uni.getStorageSync(USER_INFO_KEY);
  if (!userInfoStr) return undefined;
  
  try {
    return JSON.parse(userInfoStr) as UserInfo;
  } catch (error) {
    console.error("解析用户信息失败", error);
    return undefined;
  }
};

export const setUserInfo = (userInfo: UserInfo): void => {
  uni.setStorageSync(USER_INFO_KEY, JSON.stringify(userInfo));
};

// 权限
export const getPermissions = (): string[] => {
  const permsStr = uni.getStorageSync(PERMISSIONS_KEY);
  if (!permsStr) return [];
  
  try {
    return JSON.parse(permsStr) as string[];
  } catch (error) {
    console.error("解析权限信息失败", error);
    return [];
  }
};

export const setPermissions = (permissions: string[]): void => {
  uni.setStorageSync(PERMISSIONS_KEY, JSON.stringify(permissions));
};

// 清除所有缓存
export const clearAll = (): void => {
  uni.removeStorageSync(TOKEN_KEY);
  uni.removeStorageSync(REFRESH_TOKEN_KEY);
  uni.removeStorageSync(TOKEN_EXPIRES_AT_KEY);
  uni.removeStorageSync(USER_INFO_KEY);
  uni.removeStorageSync(PERMISSIONS_KEY);
};

// 清除令牌相关缓存
export const clearTokens = (): void => {
  uni.removeStorageSync(TOKEN_KEY);
  uni.removeStorageSync(REFRESH_TOKEN_KEY);
  uni.removeStorageSync(TOKEN_EXPIRES_AT_KEY);
};
```

## 🤔 什么时候用 `function`？

只有少数情况下用 `function`：

1. **需要函数提升时**（但通常不推荐）
2. **需要 `arguments`对象时**
3. **需要命名函数用于调试时**

```
// 特殊情况：需要命名函数用于调试
export const complexFunction = function processUserData(user: UserInfo) {
  // 在调用栈中显示 processUserData
  // 但通常可以使用 console.trace() 替代
  return user;
};
```

## 🎯 最佳实践建议

1. **导出函数时**：优先使用 `const`箭头函数
2. **模块内部函数**：可以使用 `function`声明
3. **类方法**：使用类方法语法
4. **React/Vue 组件**：使用箭头函数绑定 `this`

```
// 模块示例
const internalHelper = (): void => {
  // 模块内部私有函数
};

export const publicApi = (): string => {
  // 导出的公共 API
  internalHelper();
  return "result";
};
```

## 🔄 如果你现有的代码是 function 风格

如果你现有的代码已经是 `function`风格，可以选择：

1. **逐步迁移**：新代码用 `const`，旧代码保持 `function`
2. **统一风格**：全部改为 `const`风格
3. **保持现状**：如果团队约定用 `function`，可以保持

**推荐选择 1 或 2**，因为 `const`箭头函数是现代 JavaScript 的标准写法。

所以，对于你的 `getTokenExpiresAt`函数，建议改为：

```
// ✅ 推荐写法
export const getTokenExpiresAt = (): number => {
  const expiresAt = uni.getStorageSync(TOKEN_EXPIRES_AT);
  if (!expiresAt) return 0;
  
  const num = Number(expiresAt);
  return isNaN(num) ? 0 : num;
};
```

这样修改后，代码更简洁、更安全，也符合现代 TypeScript 的最佳实践。
