现在关键是**刷新令牌请求的参数配置**。让我给你正确的配置：

## 🎯 刷新令牌请求的正确参数配置

### 1. 前端请求参数（**关键！**）

```
refreshToken(refreshToken: string): Promise<LoginResult> {
  const clientId = getClientId() || CLIENT_CONFIG.CLIENT_ID;
  console.log("刷新令牌使用客户端ID:", clientId);
  const basicAuth = CLIENT_CONFIG.getBASIC_AUTH();
  console.log("动态生成的认证头:", basicAuth);

  // ✅ 正确参数：使用 URLSearchParams
  const formData = new URLSearchParams();
  formData.append('grant_type', 'refresh_token');           // 必须是 refresh_token
  formData.append('refresh_token', refreshToken);           // 小写+下划线
  formData.append('client_id', clientId);                   // 可选，但建议加上

  return request<LoginResult>({
    url: `${AUTH_LOGIN_URL}/oauth2/token`,
    method: "POST",
    data: formData.toString(),  // 重要：要转换成字符串
    header: {
      "Content-Type": "application/x-www-form-urlencoded",  // 必须
      "Authorization": basicAuth,                           // Basic认证
      skipAuth: true, // 跳过认证，因为还没有有效的access_token
    },
  });
}
```

### 2. 对比两种模式的参数差异

| 参数              | 微信登录模式      | 刷新令牌模式        | 说明         |
| ----------------- | ----------------- | ------------------- | ------------ |
| **grant_type**    | `wechat_mini_app` | `refresh_token`✅    | **完全不同** |
| **code**          | 微信code          | 不传                | 微信登录特有 |
| **refresh_token** | 不传              | 要传的refresh_token | 刷新令牌特有 |
| **client_id**     | 客户端ID          | 客户端ID            | 两者都传     |

### 5. 后端需要支持刷新令牌模式

确保你的 `AuthorizationServerConfig`包含：

```
@Configuration
@EnableWebSecurity
public class AuthorizationServerConfig {
    
    @Bean
    @Order(1)
    public SecurityFilterChain authorizationServerSecurityFilterChain(HttpSecurity http)
            throws Exception {
        OAuth2AuthorizationServerConfiguration.applyDefaultSecurity(http);
        
        http
            // 支持刷新令牌
            .oauth2ResourceServer(oauth2 -> oauth2.jwt(Customizer.withDefaults()))
            .exceptionHandling(exceptions -> exceptions
                .authenticationEntryPoint(new LoginUrlAuthenticationEntryPoint("/login"))
            );
        
        return http.build();
    }
}
```

### 6. 常见错误排查

| 错误                       | 原因                    | 解决方案                              |
| -------------------------- | ----------------------- | ------------------------------------- |
| **unsupported_grant_type** | grant_type 不对         | 必须用 `refresh_token`                |
| **invalid_grant**          | refresh_token 无效/过期 | 重新登录获取新的                      |
| **invalid_client**         | 客户端认证失败          | 检查 Basic Auth                       |
| **缺少 refresh_token**     | 参数名不对              | 用 `refresh_token`不是 `refreshToken` |
