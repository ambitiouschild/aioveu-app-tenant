# ✅ 一句话结论（先拍死）

> ✅ 支付成功后，前端轮询的是「订单状态（Order Status）」
>
> ❌ 不是轮询支付订单状态（Payment Status）
>
> ✅ 支付单只负责“钱到没到”
>
> ✅ 订单状态才是“业务真理源”

---

# 二、为什么不能只轮询「支付订单状态」？

## ❌ 支付单（pay / payment）的局限性

| 问题                | 原因                     |
| ------------------- | ------------------------ |
| 支付成功 ≠ 业务完成 | 钱到了，但订单可能未更新 |
| 支付单可能被拆单    | 一笔订单对应多笔支付     |
| 支付单可能异步回调  | 微信先扣款，再通知你     |
| 支付单可能补单      | 掉单后人工补录           |

👉 支付单是“财务凭证”，不是“业务结果”

---

## ✅ 订单（Order）才是业务真理源

| 订单状态 | 含义                  |
| -------- | --------------------- |
| UNPAID   | 还没付钱              |
| PAID     | ✅ 钱到了 + 订单已更新 |
| SHIPPED  | 已发货                |
| COMPLETE | 已完成                |
| CLOSED   | 已关闭                |

👉 只有订单状态变成 `PAID`，才代表：
• ✅ 微信扣款成功

• ✅ 系统已处理回调

• ✅ 库存已扣减

• ✅ 优惠券已核销

• ✅ 积分已发放

## ✅ 标准轮询流程（你这套系统应该用这个）

```
1️⃣ 调起微信支付
        ↓
2️⃣ 微信 success 回调
        ↓
3️⃣ 前端开始轮询【订单详情】
        ↓
4️⃣ 后端处理微信回调
        ↓
5️⃣ 订单状态变为 PAID
        ↓
6️⃣ 前端轮询命中 PAID
        ↓
7️⃣ 跳转支付成功页 ✅
```

## ✅✅ 正确写法（最终版）

### ✅ 轮询订单状态

```ts
const startPollingOrderStatus = (orderSn: string) => {
  let times = 0;
  const timer = setInterval(async () => {
    times++;
    try {
      const res = await getOmsOrderDetail(orderSn, channel.value);

      console.log(`🔁 第 ${times} 次轮询订单状态`, res.status);

      // ✅ 订单已支付（唯一真理）
      if (res.status === OrderStatusEnum.PAID) {
        clearInterval(timer);
        uni.hideLoading();
        uni.showToast({ title: "支付成功", icon: "success" });

        setTimeout(() => {
          uni.redirectTo({
            url: `/packageD/pages/money/paySuccess?orderSn=${orderSn}`,
          });
        }, 1500);
        return;
      }

      // ❌ 订单关闭 / 取消
      if (
        res.status === OrderStatusEnum.CLOSED ||
        res.status === OrderStatusEnum.CANCELED
      ) {
        clearInterval(timer);
        uni.hideLoading();
        uni.showModal({
          title: "支付失败",
          content: "订单已关闭或取消",
          showCancel: false,
        });
        return;
      }

      // ⏳ 最多轮询 30 秒
      if (times > 15) {
        clearInterval(timer);
        uni.hideLoading();
        uni.showModal({
          title: "支付结果未知",
          content: "请稍后查看订单状态或联系客服",
          showCancel: false,
        });
      }
    } catch (e) {
      console.error("轮询订单状态失败", e);
    }
  }, 2000);
};
```
