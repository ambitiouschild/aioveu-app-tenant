## ② responseInterceptor 做了什么？

```ts
const responseInterceptor = (response: any) => {
  const resData = response.data;

  if (response.statusCode === 401) {
    return Promise.reject({ code: "AUTH_ERROR", message: "认证失败" });
  }

  if (resData.code === ResultCodeEnum.TOKEN_INVALID || resData.code === "A0230") {
    handleTokenExpiredSync();
    return Promise.reject({ code: resData.code, message: resData.msg });
  }

  // ✅ 成功时必须包一层 Promise
  return Promise.resolve(resData.data);
};
```

✅ 注意这里：
• ❌ 没有返回 `resData`

• ✅ 返回的是 `Promise.resolve(resData.data)`

• ✅ `resData.data` 就是 后端 `data` 字段

## ③ request 里怎么消费这个拦截器结果？

```ts
const interceptorResult = responseInterceptor(responseWithConfig);

if (interceptorResult && interceptorResult.then) {
  interceptorResult.then(resolve).catch(reject);
  return;
}
```

### 这里发生了什么？

| 表达式              | 实际值                               |
| ------------------- | ------------------------------------ |
| `interceptorResult` | `Promise.resolve(resData.data)`      |
| `.then(resolve)`    | ✅ 把 `resData.data` 传给外层 Promise |
| `return;`           | ✅ 后面代码永远不会执行               |

# 三、那 `<T>` 到底是什么？

### 你的函数签名：

```ts
export default async function request<T>(options): Promise<T>
```

### 实际使用：

```ts
interface PayResp {
  orderSn: string;
  paymentChannel: string;
  paymentMethod: string;
  prepayId: string;
  payParams: {
    appId: string;
    timeStamp: string;
    nonceStr: string;
    package: string;
    signType: string;
    paySign: string;
  };
}

const res = await pay<PayResp>({ ... });
```

✅ T = 后端 Result.data 的 TS 类型
✅ request 不关心 T 的内容
✅ request 只负责“把 T 还给你”

# 四、失败时的返回结构（你也很关心）

### reject 的结构是统一的：

```ts
{
  code: string;   // TOKEN_INVALID | AUTH_ERROR | UNKNOWN
  message: string;
}
```

### 页面感知方式：

```ts
try {
  const res = await pay(...)
} catch (err) {
  err.code
  err.message
}
```

✅ 页面永远不处理 code === 200
✅ 页面只关心 success / error
