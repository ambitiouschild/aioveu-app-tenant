## 二、为什么支付页**必须展示 `orderSn`？**

### ✅ 1️⃣ 用户只认“订单号”

• 用户截图

• 找客服

• 开发查问题


👉 没人会说：“我那个 `PAY20260101xxx` 没付钱”
👉 所有人只会说：“我订单 `ORD20260101xxx` 有问题”

---

### ✅ 2️⃣ 降低客诉率（非常重要）

支付页展示：

```

支付金额：¥99.00
```

✅ 用户一眼确认
✅ 错付率大幅下降
✅ 客服压力减半

## 四、你现在的代码“正确姿势” ✅

### ✅ 页面 data

```js
data() {
  return {
    orderSn: '',      // ✅ 展示给用户
    paymentNo: '',    // ✅ 用来干活
    paymentAmount: 0
  }
}
```

---

### ✅ onLoad 接收参数

```js
onLoad(options) {
  this.orderSn = options.orderSn
  this.paymentAmount = options.paymentAmount

  // ✅ 用 orderSn 换 paymentNo
  this.createPayOrder()
}
```

---

### ✅ 创建支付单（关键）

```js
async createPayOrder() {
  const res = await createOrder({
    orderSn: this.orderSn,
    paymentAmount: this.paymentAmount
  })

  if (res.code !== 0) {
    throw new Error('创建支付单失败')
  }

  // ✅ 拿到 paymentNo，但不展示
  this.paymentNo = res.data
}
```

---

### ✅ 调起支付

```js
async onPay() {
  await payApi.pay({
    paymentNo: this.paymentNo
  })
}
```
